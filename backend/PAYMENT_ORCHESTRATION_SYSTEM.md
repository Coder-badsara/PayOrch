## 1. Project Overview & Goals

### What We Are Building

A **production-ready Payment Orchestration Layer** that acts as an abstraction layer above multiple payment gateways (Razorpay, Stripe, PayU, UPI). It intelligently routes transactions, monitors gateway health in real-time, fails over automatically in under 2 seconds, ensures exactly-once payment processing via idempotency, manages full transaction lifecycle through a state machine, and ingests + deduplicates webhooks from all connected gateways.

### Core Capabilities

| Capability | Detail |
|---|---|
| Multi-gateway support | Razorpay, Stripe, PayU, UPI (simulated adapters) |
| Intelligent routing | Rule-based + health-score-weighted selection |
| Failover speed | < 2 seconds end-to-end on gateway failure |
| Idempotency | Per-payment idempotency key with atomic DB lock |
| State machine | 11-state lifecycle with strict transition guards |
| Webhook pipeline | Ingestion → verify signature → deduplicate → reconcile |
| Observability | Structured JSON logs + Prometheus metrics + OpenTelemetry traces |

## 6. Part 1 — Payment Orchestration Architecture & Multi-Gateway Routing

### 6.1 Gateway Adapter Interface — `src/gateways/base.gateway.ts`

> **Copilot:** Generate this complete abstract class with all method signatures and JSDoc.

```typescript
// Every gateway adapter MUST extend this class.
// It defines the standard interface for all gateway operations.

export interface CreateOrderParams {
  amount: number;           // in smallest currency unit
  currency: string;
  receipt: string;          // internal idempotency reference
  notes?: Record<string, string>;
}

export interface CreateOrderResult {
  gatewayOrderId: string;
  gatewayPaymentId?: string;
  checkoutPayload: Record<string, unknown>; // data sent to frontend SDK
  rawResponse: unknown;
}

export interface CapturePaymentParams {
  gatewayOrderId: string;
  gatewayPaymentId: string;
  amount: number;
  currency: string;
}

export interface RefundPaymentParams {
  gatewayPaymentId: string;
  amount: number;
  reason?: string;
  notes?: Record<string, string>;
}

export interface GatewayPaymentStatus {
  gatewayStatus: string;
  normalizedStatus: TransactionStatus;
  amount: number;
  currency: string;
  rawResponse: unknown;
}

export interface HealthCheckResult {
  isHealthy: boolean;
  latencyMs: number;
  error?: string;
}

export abstract class BaseGateway {
  abstract readonly name: GatewayName;

  /** Create an order/intent on the gateway */
  abstract createOrder(params: CreateOrderParams): Promise<CreateOrderResult>;

  /** Capture a previously authorized payment */
  abstract capturePayment(params: CapturePaymentParams): Promise<void>;

  /** Initiate a refund */
  abstract refundPayment(params: RefundPaymentParams): Promise<string>; // returns gatewayRefundId

  /** Fetch current status of a payment from the gateway */
  abstract getPaymentStatus(gatewayPaymentId: string): Promise<GatewayPaymentStatus>;

  /** Lightweight health check (ping or balance fetch) */
  abstract healthCheck(): Promise<HealthCheckResult>;

  /** Verify webhook signature */
  abstract verifyWebhookSignature(
    rawBody: Buffer,
    signature: string,
    secret: string,
  ): boolean;

  /** Normalize gateway-specific webhook event to a standard shape */
  abstract normalizeWebhookEvent(rawPayload: unknown): NormalizedWebhookEvent;
}
```

## 7. Part 2 — Gateway Health Monitoring & Intelligent Routing Algorithm

### 7.1 Health Monitor Service — `src/orchestrator/health-monitor.service.ts`

> **Copilot:** Generate this service completely. It must:

1. Run on a `setInterval` of `HEALTH_CHECK_INTERVAL_MS` (15s default).
2. Call `gateway.healthCheck()` for each registered gateway in **parallel** using `Promise.allSettled`.
3. Maintain a **sliding window** of the last 100 calls per gateway (stored in Redis as a list).
4. Compute:
   - `successRate` = successful calls / total calls in window
   - `avgLatencyMs` = mean of latency values in window
   - `p95LatencyMs` = 95th percentile latency
5. Classify gateway status:
   ```
   successRate >= HEALTH_DEGRADED_THRESHOLD  → HEALTHY
   successRate >= HEALTH_UNHEALTHY_THRESHOLD → DEGRADED
   successRate <  HEALTH_UNHEALTHY_THRESHOLD → UNHEALTHY
   ```
6. **Circuit breaker logic:**
   - If status becomes `UNHEALTHY` for 3 consecutive checks → set `CIRCUIT_OPEN`
   - Circuit stays open for `CIRCUIT_RESET_TIMEOUT_MS` (default 60s)
   - After timeout, attempt one health check → if passes, move to `DEGRADED`
7. Persist health state to `GatewayHealth` table and log to `GatewayHealthLog`.
8. Emit a `gateway:health:changed` event when status changes (use Node `EventEmitter`).
9. Expose `getHealthSnapshot(): Record<GatewayName, GatewayHealth>` for the router.
10. Expose Prometheus gauges:
    - `gateway_success_rate{gateway}` (Gauge)
    - `gateway_avg_latency_ms{gateway}` (Gauge)
    - `gateway_circuit_state{gateway}` (Gauge: 0=closed, 1=open)

**Redis key pattern for sliding window:**
```
health:window:{GATEWAY_NAME} → Redis LIST (LPUSH, LTRIM to 100)
Each element: JSON { success: boolean, latencyMs: number, ts: number }
```
